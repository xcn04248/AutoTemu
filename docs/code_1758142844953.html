How to release product?

Last update time: 2025-09-09 11:37:06

Overview

The Product API helps sellers manage their product catalog at scale. Using the Product API, a seller can create products.

The following is an explanation of the field values and API calls involved in product publishing. We hope this document will help you to publish products more efficiently.

API Interfaces for Product Release

The following are the interfaces involved in product release:

API Name Interface Overview
bg.local.goods.cats.getGet the complete category of temu
bg.local.goods.template.getObtain attribute/variant data corresponding to temu category
bg.local.goods.spec.id.getUsed to generate custom variant specification IDs
bg.local.goods.size.element.getWhether to fill in the size chart and requirements for obtaining categories
bg.local.goods.image.uploadUpload product images
bg.local.goods.gallery.signature.getUpload product videos, files, and images
bg.local.goods.compliance.property.checkVerify whether the product attributes comply with the site's sales rules
bg.local.goods.tax.code.getObtain the product tax code
bg.local.goods.out.sn.checkVerify if the external product code and SKU code are duplicated
bg.local.goods.sku.out.sn.checkVerify if the external product code and SKU code are duplicated
bg.freight.template.list.queryGet the seller's set product shipping template
bg.local.goods.brand.trademark.getObtain the brand information registered by the seller
temu.local.goods.sku.net.content.unit.queryObtain SKU net content unit information
temu.local.goods.illegal.vocabulary.checkVerify whether there are any violations in the product name, product description, and other information
bg.local.goods.compliance.rules.getUsed to query product compliance and governance attribute requirements
bg.local.goods.compliance.extra.template.getUsed to query product compliance and governance attribute requirements
bg.local.goods.compliance.info.fill.list.queryUsed to obtain product compliance information and fill in
bg.local.goods.addCreate a new product

goodsBasic Required

goodsName Required

  • How to fill in: User inputs
  • Validation Rules:
    • Only supports English letters, numbers, and symbols
    • Does not support decorative characters: ~ ! * $ ? _ ~ { } # < > | * ; ^ ¬ ¦
    • Does not support high ASCII characters type 1, such as ®, ©, ™, etc.
    • Character count: Within 500 characters
Precautions

Use temu.local.goods.illegal.vocabulary.check to check for any violations in product information to avoid affecting sales.

catId Required

  • How to fill in: Retrieve from other APIs
  • API Reference: bg.local.goods.cats.get
  • Helper API: bg.local.goods.category.recommend will help you quickly match temu categories based on product information
Important Notes
  • Entering parentCatId = 0 retrieves all available first-level categories for posting
  • To obtain leaf categories: Recursively call this interface, entering parentCatId with the catId selected from the previous call's results to get the next level of category ID until the leaf categories are reached
  • When entering a leaf category, the interface returns empty
  • Ensure to use the most specific (leaf) category ID for posting products
  • catType is an important variable for judgement whether category is an Apparel (catType=0) or Non-Apparel (catType=1) one, it will effect at the image required shape
  • Some categories on different sites are not sellable, only the categories with availableStatus=0 need to be obtained

goodsProperty Required

  • How to fill in: Retrieve from other APIs
  • API Reference: bg.local.goods.template.get
  • Description: This is a very important and complex piece of information, and it takes more time to process it when entering it
Important Notes
  • Determine whether the field isSale=false is a normal attribute or isSale=true is a variant attribute
  • Required attributes must be passed in, judging by bg.local.goods.template.get - required=True
  • There is a parent-child relationship attribute. When the parent attribute selects a value, the child attribute must be passed in
  • showType = 0: Parent attribute
  • showType = 1: Child attribute
  • Child attributes appear based on the controlType of the parent attribute
  • If controlType = 0, showCondition will indicate the conditions under which child attributes are triggered based on parent attribute values
  • If controlType is 1, 3, or 16, the templatePropertyValueParentList will determine when child attributes are triggered
  • Attributes that have units should include both valueUnitId and valueUnit when available in the valueUnitList
Verification

Use bg.local.goods.compliance.property.check to check for any violations in product information to avoid affecting sales.

goodsOriginInfo Optional

Size Chart Information

Description: For some categories, corresponding size chart information needs to be set. Currently, it is not mandatory when products are released, but if it is not supplemented before sale, sales will be restricted.

API Reference: bg.local.goods.size.element.get to determine if size chart is necessary for the category

goodsSizeImage Optional

  • Description: Size chart image
  • API Reference: bg.local.goods.size.element.get to determine if size chart is necessary for the category
  • Recommendations: The image should include the following information: all selected size, product parameters, use size units
  • Conversion: Convert images from bg.local.goods.image.upload to Temu

Specifications:

  • Quantity: 1 piece
  • Format: jpeg, jpg, png
  • Aspect Ratio (width:height): ≥1:3
  • Width: ≥800px
  • Height: ≥800px
  • Size: ≤3 MB

skuList Required

Description: This amount information is the amount that the merchant plans to sell in TEMU. If there are subsequent orders generated, this amount information will be used to calculate the settlement amount with the merchant.

specIdList Required

Description: This is a field used to distinguish variants, such as color, size, etc. Please note that there are two ways to set this information for different categories on the platform. Setting this information is often prone to errors and requires special attention.

Category Template System
  • Each category has a set of attribute templates, which can be retrieved via the bg.local.goods.template.get interface
  • Sales attributes are further classified based on whether customization is allowed

Customization Rules:

  • If the category allows user-defined attributes, inputMaxSpecNum > 0 is returned
  • If the category only allows selecting predefined attributes, inputMaxSpecNum = 0 is returned. In this case, you should only select attributes marked with required = true in the goodsSpecProperties object, and pick a few values
  • Custom attributes are allowed only when inputMaxSpecNum > 0
  • In this case, you can choose a parentSpec from userInputParentSpecList, and use bg.local.goods.spec.id.get to generate specification values
  • It's possible for inputMaxSpecNum > 0 and goodsSpecProperties != [] to occur at the same time, which means the category allows both custom attributes and predefined attribute values
  • singleSpecValueNum defines the maximum number of variants that can be created
Important Note

bg.local.goods.spec.id.get generates specification values, which should be passed as input parameters in the skuList[*].specIdList field of the goods.add method.

outSkuSn Optional

Description: If a merchant operates on multiple platforms, they usually use this field to identify products associated with other platforms. This field is very important for sellers and will affect the subsequent inventory and order product information association.

Specifications:

  • Characters: English letters/numbers/symbols
  • Number of characters: ≤100
  • Must not be the same as any other SKU/SPU ID in the same store
Verification

bg.local.goods.sku.out.sn.check can be used to verify whether the seller's products have duplicate goods codes.

Physical Dimensions Required

All physical dimension fields (weight, weightUnit, length, width, height, volumeUnit) are required and different sites have corresponding requirements for this information.

images Required

Description: Identify whether it belongs to the clothing category through the bg.local.goods.cats.get - catType, as different types have differences in size ratios.

Apparel Carousel Images:

  • Aspect Ratio: 3:4
  • Width: ≥1340px
  • Height: ≥1785px

Non-Apparel Carousel Images:

  • Aspect Ratio: 1:1
  • Width: ≥800px
  • Height: ≥800px

Common Specifications:

  • Size: ≤3 MB
  • Format: jpeg, jpg, png
Note

It is necessary to upload the images to TEMU in advance, and using bg.local.goods.image.upload will facilitate users to handle the corresponding image proportions and sizes in a portable manner.

External Product Information

externalProductType Optional

Values: 1=EAN, 2=UPC, 3=ISBN, 4=GTIN-14

externalProductId Optional

Description: When setting externalProductType, this field is mandatory and the platform will verify the authenticity of the entered externalProductId. Different categories have mandatory requirements.

Note: bg.local.goods.template.get - Added parameters [additionalInfo] and [needIsbn] to determine if a mandatory ISBN code is required.

multiplePackage Optional

skuClassification Optional

Values: 1-Single Product, 2-Combination, 3-Mixed

numberOfPieces Optional

Description: When skuClassification is set to 1, this field defaults to 1, while other types are set by the merchant. The value of the "Count" field should be between [1,100].

originNetContentNumber Optional

Description: For products consumed by volume, weight, length and area, this field is used to describe the quantity of the corresponding physical units in the SKU to calculate the unit price, such as $5.94/kg. Unit count can contain 3 decimal places, and any decimal places exceeding 3 will be rounded off. For skuClassification=2, only then will this field be displayed for users to set.

netContentUnitCode Optional

Description: The unit corresponding to Unit count. This can be weight, volume, length or area. For skuClassification=2, only then will this field be displayed for users to set.

API Reference: temu.local.goods.sku.net.content.unit.query to obtain specific values

Pricing & Inventory Configuration

basePrice Required

Note: Different sites have corresponding requirements for currency information.

listPrice Optional

Description: Recommended retail price (must be ≥ basePrice)

listPriceType Optional

  • If the parameter is set to 1, it is understood that the seller does not have a market price, and there is no need to provide a value for listPrice (even if a value is provided, it will be ignored)
  • If the parameter is left empty or set to 0, the listPrice field is required. Failure to provide listPrice will result in an error

quantity Required

Description: This is the inventory information corresponding to the product, with a setting range of [0, 999999].

referenceLink Optional

Description: Provide links to products sold on other platforms. It is recommended to provide this feature for merchants to set up, which will help with their sales process.

goodsTrademark Optional

brandId Optional

  • How to fill in: Retrieve from other APIs
  • Description: The parameters returned by bg.local.goods.template.get with refpid=1960 are all brand attributes
Note

Please note that the brand information of each seller is different, and it needs to be reviewed and approved in advance in the seller center before the product can be published for use.

trademarkId Optional

Description: The specific information is returned through the interface, and it should be noted that this data is different for each merchant. It is necessary to apply in advance to the seller center.

taxCodeInfo Optional

  • How to fill in: Retrieve from other APIs
  • API Reference: bg.local.goods.tax.code.get
  • Description: Different categories have corresponding tax rate codes. It is recommended to obtain them by referencing the corresponding category at the time of shipment

goodsVehiclePropertyRelation Optional

secondHand Optional

Description: If a merchant needs to sell second-hand goods, they need to contact Temu's internal operations in advance to open it. Only stores that have applied for approval can publish second-hand goods. The merchant can determine whether they meet the conditions for publishing second-hand goods based on BG. If they meet the requirements, bg.local.goods.cats.get - secondHandCategory=true will return the corresponding results.

secondHandGoods Optional

Description: If selling second-hand goods, fill in true by default

level Optional

Values: 1-usedLikeNew, 2-usedVeryGood, 3-usedGood, 4-UsedAcceptable
